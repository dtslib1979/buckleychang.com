## 1. 핵심 포지셔닝

### 한 문장 정의

> **Buckley Chang = AI 시대 1인 미디어 창작자의 전담 관제탑**
> 직접 다 하는 게 아니라, 이 세계를 가장 잘 알기 때문에 어디로 보내야 하는지 판단하는 역할.

### 왜 "관제탑"인가

| 일반 전문가 | 한계 | Buckley |
|------------|------|---------|
| 일반 세무사 | AI 수익 구조 모름. YouTube/Patreon/Super Thanks 구분 못 함 | 이 세계를 살아봤기 때문에 수익 구조를 안다 |
| 일반 변호사 | 저작권 알지만 콘텐츠 IP가 자산이 되는 타이밍 모름 | 전환 시점을 프로토콜로 정의했다 |
| 일반 변리사 | 특허 알지만 AI 생성물 저작권 경계선 못 그음 | AI 크리에이터 실무를 안다 |
| 일반 MCN | 서비스 제공하지만 수익을 가져감 | 수익을 안 가져감. 라우팅만 한다 |

### "Reality Interface"의 진짜 의미

```
크리에이터 (창작의 세계)
       │
       ▼
  Buckley Chang (Reality Interface)
       │
       ├── 이 문제는 세무 → CPA (직접)
       ├── 이 문제는 저작권 → 변리사 (라우팅)
       ├── 이 문제는 법무 → 변호사 (라우팅)
       └── 이 문제는 법인 → Buckley + 변호사 (협업)
```

Reality Interface = **창작의 세계와 비즈니스 현실 사이의 번역기**

## 2. 타겟 클라이언트 프로파일

### 대상: AI 활용 1인 미디어 크리에이터

| 특성 | 설명 |
|------|------|
| **플랫폼** | YouTube 주력, 부가적으로 Patreon/Gumroad/Newsletter |
| **수익 범위** | 월 100만~1000만+ (GREEN→YELLOW 전환 구간) |
| **AI 활용** | 영상 제작, 음성 합성, 자막, 썸네일, 대본 등에 AI 도구 사용 |
| **고민** | "이거 사업자 내야 하나?", "해외 수익 신고 어떻게?", "AI로 만든 건 내 저작물인가?" |
| **현재 상태** | 혼자서 모든 걸 하고 있음. 세금은 5월에 급하게 처리. 법률 이슈는 모르고 지나감 |

### 대상이 아닌 사람

- MCN 소속 크리에이터 (이미 서비스 받고 있음)
- 대형 미디어 기업 (규모가 다름)
- 비 AI 전통 콘텐츠 크리에이터 (AI 특화 가치가 줄어듦)

## 3. 서비스 포털의 5가지 전환점 (Moments)

크리에이터가 겪는 전환점을 입구로 사용:

### Moment 1: 첫 해외 수익이 들어왔다

```
상황: YouTube AdSense에서 USD가 들어왔다
질문: 원천징수? 환전? 신고?
Zone: YELLOW 진입
담당: Buckley 직접
```

### Moment 2: 내 콘텐츠를 누군가가 무단 사용했다

```
상황: AI로 만든 캐릭터/음악/영상이 도용당했다
질문: 상표? 저작권? AI 생성물 기준은?
Zone: YELLOW → RED
담당: 변리사/변호사 라우팅
핵심 이슈: 2026년 기준 AI 단독 생성물은 저작권 불가 (미 대법원)
```

### Moment 3: 수익이 월 300만원을 넘었다

```
상황: 종합소득세가 갑자기 커졌다
질문: 사업자 언제? 간이과세? 일반과세?
Zone: GREEN → YELLOW
담당: Buckley 직접
핵심: 업종코드 940306 vs 921505 판단
```

### Moment 4: 브랜드 딜이 들어왔다

```
상황: 기업에서 협찬/광고 제안이 왔다
질문: 계약서 검토. 세금계산서. 전속 조항. 수익 인식 시점.
Zone: YELLOW
담당: Buckley (세무) + 변호사 (계약) 라우팅
```

### Moment 5: 법인을 만들고 싶다

```
상황: 규모가 커져서 법인 전환을 고려한다
질문: 지분 구조. IP 이전. 절세 설계. 대표이사 급여.
Zone: RED
담당: Buckley + 변호사 협업
```

## 4. Traffic Signal Protocol — 서비스의 핵심 엔진

### Zone 정의

| Zone | 조건 | Buckley 역할 | 행동 |
|------|------|-------------|------|
| 🟢 GREEN | 개인, 초기 수익, 국내만 | 관여 안 함 | 가이드/문서 제공 |
| 🟡 YELLOW | 트리거 1개+ 충족 | 자동 호출 | 진단 → 라우팅 |
| 🔴 RED | 법인/지분/IP분리/매각 | 전권 통제 | 직접 + 파트너 협업 |

### Trigger Thresholds

| 트리거 | 임계값 | 근거 |
|--------|--------|------|
| 월 수익 | ₩3,000,000 초과 | 간이과세 기준, 실질적 사업 규모 |
| 해외 결제 | 최초 1건 | 원천징수/환율/이중과세 이슈 발생 |
| 인력 고용 | 1명 이상 | 4대보험, 원천징수 의무 |
| IP 분리 요청 | 명시적 언급 | 자산 구조 변경 필요 |
| 법인 문의 | "법인 만들까요?" | 지분/자본/이사 구조 설계 필요 |

### 프로토콜이 왜 해자(Moat)인가

1. **경쟁사 없음** — 한국에서든 해외에서든 크리에이터용 프로토콜 기반 진단 허브는 없음
2. **자동화 가능** — 임계값이 정량적이므로 도구로 구현 가능
3. **확장 가능** — 트리거 추가/수정만으로 서비스 범위 확장
4. **리드 제너레이션** — Zone Check 도구가 자연스러운 진입점 (전환율 15-25%)
5. **서사적 가치** — "당신의 Zone을 알려드립니다"는 콘텐츠가 됨

## 5. 파트너 네트워크 전략

### "모르는 영역"이 약점이 아닌 이유

> "저는 세법 밖의 질문에 아는 척하지 않습니다.
> 대신 이 분야에서 제가 신뢰하는 사람에게 직접 연결합니다."

이 메시지가 신뢰의 근거가 됨:
- CPA가 저작권법을 조언하면 → 불신
- CPA가 "이건 제 영역이 아닙니다, 이 변리사에게 연결합니다" → 신뢰

### 파트너 카테고리

| 카테고리 | 역할 | Buckley와의 관계 |
|----------|------|-----------------|
| **세무** | 종합소득세, 부가세, 원천징수 | Buckley 직접 (CPA) |
| **법무** | 계약 검토, 법인 설립, 지분 구조 | 변호사 라우팅 |
| **IP/저작권** | 상표, 저작권, AI 생성물 판단 | 변리사 라우팅 |
| **국제** | 크로스보더 세무, 해외 법인, 이중과세 | Buckley 직접 + 해외 파트너 |

### 라우팅 프로세스

```
1. 크리에이터가 Zone Check 완료 → YELLOW/RED 진단
2. Buckley가 문제를 분류
3. 세무 → 직접 처리
4. 비세무 → "이 파트너에게 연결합니다" + 브리핑 전달
5. 파트너 처리 후 → Buckley가 전체 상황 통합 관리
```

## 6. 사이트 톤 & 보이스

### 기존 (명함)

> "CPA · Reality Interface · Parksy World"

### 전환 후 (서비스 포털)

> "저도 이 길을 걸었습니다.
> 첫 해외 수익이 들어왔을 때 뭘 해야 하는지 아무도 알려주지 않았습니다.
> 그래서 이 프로토콜을 만들었습니다."

### 톤 원칙

1. **경험에서 나온 말** — 일반 회계사의 서비스 나열이 아니라 "나도 겪었다"
2. **정직한 경계** — "이건 제 영역이 아닙니다"가 신뢰를 만듦
3. **프로토콜의 언어** — GREEN/YELLOW/RED가 자연스럽게 대화에 등장
4. **한국어 기본, 영어 지원** — 핵심 고객은 한국 크리에이터

## 7. "소설이면서 역할극"의 사이트 적용

### 서사 구조

```
일반 회계사 사이트: 서비스 목록 → 연락처
Buckley 포털: 내가 겪은 이야기 → 당신의 상황 진단 → 해결책 라우팅
```

### 각 서비스 페이지의 서사

**예시: /services/international/**

> **Chapter: 수익이 국경을 넘는 흐름이 될 때**
>
> AdSense에서 첫 USD가 들어온 날을 기억합니다.
> ₩1,345.50/USD. 환전 수수료. 원천징수 30%.
> 아무도 알려주지 않았습니다.
>
> 이것이 제가 하는 일입니다:
> - 미국-한국 조세조약 적용 여부 판단
> - 원천징수 세율 최적화 (30% → 15% 또는 0%)
> - 외화 수익 신고 방법 설계
> - 해외 플랫폼별 세금 처리 가이드
>
> [내 Zone 체크하기] [연락하기]

이 톤이 일반 CPA 사이트와의 차별점. **레포지토리가 소설이듯, 서비스 페이지도 서사.**
